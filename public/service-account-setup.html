Google Sheets Service Account Setup

This guide will walk you through setting up a Google Service Account to enable direct read/write access to your Google Sheets data from the AI Cold Call Agent application.

1 Create a Google Cloud Project

  1. Go to the Google Cloud Console
  2. Click on the project dropdown at the top of the page
  3. Click "New Project"
  4. Enter a name for your project (e.g., "AI Cold Call Agent")
  5. Click "Create"
  6. Wait for the project to be created and then select it

2 Enable the Google Sheets API

  1. In the left sidebar, navigate to "APIs & Services" > "Library"
  2. Search for "Google Sheets API"
  3. Click on "Google Sheets API" in the results
  4. Click "Enable"

3 Create a Service Account

  1. In the left sidebar, navigate to "APIs & Services" > "Credentials"
  2. Click "Create Credentials" at the top
  3. Select "Service Account"
  4. Enter a name for your service account (e.g., "ai-cold-call-service")
  5. Optionally add a description
  6. Click "Create and Continue"
  7. For the role, select "Project" > "Editor" (or a more restricted role if you prefer)
  8. Click "Continue"
  9. You can skip the "Grant users access" section for now
  10. Click "Done"

4 Generate a Service Account Key

  1. You'll be redirected to the Credentials page
  2. Find your newly created service account in the "Service Accounts" section
  3. Click on the service account email address
  4. Go to the "Keys" tab
  5. Click "Add Key" > "Create New Key"
  6. Select "JSON" as the key type
  7. Click "Create"
  8. The key file will be automatically downloaded to your computer

5 Share Your Google Sheet with the Service Account

  1. Open your Google Sheets document
  2. Click the "Share" button in the top-right corner
  3. In the "Add people" field, enter the email address of your service account (found in the client_email field of your JSON file)
  4. Make sure to give it "Editor" access
  5. Uncheck "Notify people" (since a service account can't receive emails)
  6. Click "Share"

6 Configure Your Application

  1. Rename the downloaded JSON key file to service-account.json
  2. Place this file in the root directory of your AI Cold Call Agent application (the same folder where server.js is located)
  3. Restart your server

The application is already configured to look for the service-account.json file in the root directory. If you place it there, it will be detected automatically when you restart the server.

7 Verifying Your Setup

After placing the service-account.json file and restarting your server, check the console logs. You should see a message like: 

📂 Found service-account.json in root directory
✅ Google Sheets authenticated with service account

This indicates that your service account has been successfully detected and is being used for Google Sheets authentication.

Security Recommendations

  • Never commit your service-account.json file to version control
  • Consider using environment variables in production environments
  • Restrict the service account's permissions to only what's necessary
